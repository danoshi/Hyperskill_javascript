Description

In this stage, think about the security of your data. In real life, user information may leak, so you must minimize the potential losses when data is leaked.

If you do not store personal information, then the most valuable data is passwords, because people do not like to come up with complicated and unique passwords every time. Often the same password is used for all services, including banking, which is very dangerous for a person. This is why you need to convert the password strings in your database.csv file and hash them. Hash files cannot be decoded using the secret key, which means that even the company does not know the customer passwords, which also increases data security.

Objectives

  • Your program must hash each user's password using the SHA-256 hash function for data from the database.csv file;

  • Write the resulting data with the converted password to the hash_database.csv file created for you. This file contains only one line id, nickname, password, consent to mailing.

Do not output anything to the console. The tests check only the hash_database.csv file.
